# DESIGN.md — "포근한 장난감 방" 디자인 가이드

사이트 전체 리뉴얼의 기준 문서. **메인 홈(`/index.html`)의 `<style>` 블록이 기준 구현**이니
막히면 그 코드를 그대로 참고한다. 모든 작업은 **CSS만** 고친다 (아래 "하지 말 것" 참고).

## 컨셉 — 포근한 장난감 방

지금의 유치원 파스텔 감성을 유지하되 한 단계 섬세하게 다듬는다. 카드는 아이 방 바닥에
놓인 **장난감 블록**처럼 보여야 한다: 흰 종이공작 표면에 아주 옅은 색이 스미고, 바닥면은
그 카드의 색을 짙게 만든 **유채색 단면(chunk)**이 받치고, 바닥에는 앱 대표색이 은은하게
번진 **부드러운 원거리 그림자**가 깔린다. 검정 반투명 그림자(`rgba(0,0,0,.1)`)는 쓰지
않는다 — 회색 그림자는 장난감을 플라스틱처럼 싸구려로 보이게 한다. 배경은 앱 대표색
계열의 3색 그라데이션 위에 큰 블러 글로우 두 개를 띄워 방 안의 조명 같은 깊이감을 준다.

## 공통 토큰

### 모서리 반경 단계

| 토큰 | 값 | 용도 |
|---|---|---|
| `--r-sm` | `14px` | 작은 배지·칩 |
| `--r-md` | `20px` | 버튼·패널 |
| `--r-lg` | `32px` | 앱 카드·큰 판 |
| pill | `999px` | 알약 버튼(프로필·둥근 버튼) |

### chunk색 만들기 (카드색 → 바닥 단면색)

같은 hue를 유지한 채 **명도(HSL L) −18~25%p, 채도(S) +5~10%p**.
예) 노랑 `#FFD93D`(hsl 52,100%,62%) → `#D9A800`(≈hsl 47,100%,43%).
색이 탁해 보이면 채도를 더 올리고, 형광처럼 튀면 명도를 조금 더 내린다.
앱별로 계산해 둔 값은 아래 색표를 그대로 쓴다.

### chunk 그림자 + 원거리 그림자 공식

```css
/* 기본(떠 있는) 상태 */
box-shadow:
  inset 0 0 0 1px rgba(255,255,255,.65),   /* 1px 흰 안쪽 테두리 — 종이공작 느낌 */
  0 7px 0 var(--chunk),                    /* 유채색 바닥 단면 — 장난감 블록 */
  0 16px 26px rgba(var(--accent), .18);    /* 대표색이 번진 원거리 그림자 */
```

- `--chunk`: 색표의 chunk색(hex). `--accent`: 색표의 대표색 **rgb 삼중값**(`255,217,61` 꼴).
- 작은 버튼은 chunk를 `0 5px 0`, 원거리를 `0 12px 20px rgba(...,.16)`으로 줄인다.
- 기존 코드의 `box-shadow: 0 Npx 0 rgba(0,0,0,.1)`을 발견하면 전부 이 공식으로 바꾼다.

### 카드 표면 그라데이션 (틴트 표면)

```css
background: linear-gradient(180deg, #FFFFFF 0%, var(--tint) 100%);
```

`--tint`는 대표색의 아주 옅은 톤(HSL 명도 92~96%). 색표 값 사용.
흰 단색 `#fff`였던 카드가 대상이며, 이미 색이 있는 버튼(레벨 버튼 등)은 그대로 둔다.

### 눌림 상태 (chunk가 줄며 눌리는 느낌)

```css
.카드:active {
  transform: translateY(4px) scale(.98);
  box-shadow:
    inset 0 0 0 1px rgba(255,255,255,.65),
    0 3px 0 var(--chunk),                    /* 7px → 3px: 블록이 눌린다 */
    0 9px 14px rgba(var(--accent), .12);
}
/* transition: transform .12s, box-shadow .12s; 를 기본 상태에 넣는다 */
```

작은 버튼(기본 5px)은 `translateY(3px)` + `0 2px 0`으로.

### 배경 공식 (앱별 body)

기존 배경 그라데이션의 색은 유지하되, 아래처럼 겹쳐 깊이를 준다.

```css
background:
  radial-gradient(ellipse at 15% 8%,  rgba(<밝은 대표색 rgb>, .45), transparent 60%),  /* 글로우 1 */
  radial-gradient(ellipse at 88% 88%, rgba(<보조색 rgb>, .40),     transparent 58%),  /* 글로우 2 */
  radial-gradient(rgba(255,255,255,.55) 1.2px, transparent 1.3px),                    /* 물방울 도트(선택) */
  linear-gradient(160deg, <기존 3색 그라데이션>);
background-size: auto, auto, 24px 24px, auto;
```

- 글로우는 배경 그라데이션에 이미 쓰인 색 계열에서 고른다(새 색 발명 금지).
- 도트는 매우 옅어야 한다. 콘텐츠 가독성을 해치면 도트 레이어만 뺀다.

### 등장 모션 (홈·목록 화면의 카드)

```css
@keyframes card-in {
  from { opacity: 0; transform: translateY(14px); }
  to   { opacity: 1; transform: translateY(0); }
}
.카드 { animation: card-in .35s ease-out backwards; }
/* 스태거: 형제 순서대로 0.05s 간격 */
.목록 .카드:nth-child(1) { animation-delay: .05s; }
.목록 .카드:nth-child(2) { animation-delay: .10s; }
/* ...카드 수만큼. 8개를 넘기면 그 뒤는 마지막 지연값을 같이 쓴다. */

@media (prefers-reduced-motion: reduce) {
  .카드 { animation: none; transition: none; }
}
```

주의: 카드에 이미 다른 `animation`(흔들림 피드백 등)이 있으면 클래스 우선순위로
겹치지 않게 확인한다. `prefers-reduced-motion`에서는 등장·흔들림 모두 끈다.

### 글자

- 제목은 기존처럼 **900 웨이트 + 흰 테두리 그림자**(`text-shadow: 0 2px 0 rgba(255,255,255,.85)`)
  를 유지하되, 색을 한 톤 짙게 해 가독성을 올린다 (예: `#6E4A8C` → `#5E3D7C`).
- 보조 설명 글자도 배경 대비가 4:1은 되도록 짙게 보정한다. **글자 크기는 줄이지 않는다.**

## 하지 말 것

- **선택자·클래스·id 변경 금지.** JS와 e2e가 클래스 이름에 의존한다.
- **HTML 구조 변경 금지 — CSS만 고친다.** 장식이 필요하면 `::before`/`::after`로.
- **`display`·`hidden` 의미 변경 금지.** `.app-card[hidden]{display:none}` 같은 기능 규칙은
  반드시 그대로 유지한다 (부모 설정·잠금이 이 규칙으로 동작한다).
- **레이아웃 치수를 크게 바꾸지 않는다.** 카드 너비·패딩·gap의 `clamp()` 값은 유지하거나
  ±10% 안에서만 손댄다. 터치 영역은 절대 줄이지 않는다.
- **글자 크기 축소 금지.** 아이 가독성이 우선이다.
- 외부 폰트·이미지·CDN 금지 (기존 절대 규칙과 동일).
- 검정 반투명 chunk(`rgba(0,0,0,.n)`)로 되돌아가지 않기.

## 적용 후 검증

1. 폰 세로(390×844)와 패드(1180×820)에서 스크린샷을 찍어 직접 본다
   (Playwright는 `/opt/node22/lib/node_modules/playwright`).
2. 콘솔 오류 0, 해당 앱 validate + e2e 통과 후에만 커밋 (CLAUDE.md 절차).
3. 확인 포인트: 잘린 요소 없음, chunk색이 배경과 섞여 안 보이는 카드 없음,
   글자 대비 충분, 그림자가 과해 지저분하지 않음, 눌림 피드백이 자연스러움.

## 가로모드 우선 (태블릿 가로가 기본 사용 환경)

아이가 주로 **패드 가로**로 쓴다. 모든 화면은 가로모드에서 가장 보기 좋아야 한다.

- **기준 화면**: ① 패드 가로 1180×820 (1차 기준 — 여기서 예뻐야 함) ② 폰 가로 844×390 (잘림 없이)
  ③ 폰 세로 390×844 (기존 레이아웃 유지 — 건드리지 않는다).
- **재배치 원칙**: 세로로 길게 쌓인 화면은 가로에서 **2단 배치**로 —
  왼쪽에 주 콘텐츠(캔버스·보드·그림), 오른쪽에 조작(보기 버튼·팔레트·목록·진행).
  `@media (orientation: landscape)` 안에서 `grid-template-areas`로만 재배치한다 (HTML 무변경, 선례: pet.css 끝부분).
- **목록·메뉴**는 가로에서 열 수를 늘린다 (2열 → 3~4열).
- **잘림 금지**: 어떤 화면도 가로에서 카드/판이 화면 밖으로 나가거나 안 보이는 스크롤이 생기면 안 된다
  (긴 목록 화면의 자연스러운 세로 스크롤은 예외).
- 판정 자동화: `getBoundingClientRect` + `scrollHeight<=clientHeight` 검사로 세 화면 전부 확인 (선례:
  펫 방 검증 스크립트).
- 기존 규칙 그대로: `[hidden]`·display 의미, 터치 크기, clamp(), reduced-motion. 세로모드는 무변경.
