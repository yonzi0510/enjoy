# TEAM.md — 에이전트 팀 운영 지침서

이 저장소는 **역할별 에이전트 팀**으로 작업한다. 새 세션에서도 이 문서대로 팀을 꾸리면
이전과 동일한 방식·품질로 작업할 수 있다. (규칙의 원본: `CLAUDE.md`, 디자인 기준: `DESIGN.md`)

## 팀 구성과 역할

| 팀 | 역할 | 산출물 |
|---|---|---|
| 💡 **아이디어팀** | 새 학습 게임 구상 **+ 기존 앱 디벨롭(개선) 제안 — 반복 수행** | 아이디어 제안서: 이름·학습 목표·놀이 방법·구현 방식·난이도·점수(교육효과/5세 적합성/재미 지속성/구현 용이성 각 10점)·추천 순위표 |
| 🎨 **디자인팀** | 디자인 시스템·일러스트·캐릭터 (예: 장난감 방 UI, 씬 그림책 리디자인, 펫 SVG 캐릭터) | DESIGN.md 갱신, 기준 구현, 데모/스크린샷 자기 검수 |
| 🔧 **기술팀(제작)** | 승인된 아이템 구현 — 새 앱 구축 또는 기존 앱 확장 | 코드 + validate/e2e 도구 + 스크린샷 검증 |
| 📈 **개발팀(콘텐츠 확장)** | 기존 섹션의 종류·횟수 늘리기 (**최소 30개 기준**, 기존 id·형식 유지·추가만) | 데이터 확장 + 검증기 하한 강화(회귀 방지) |
| 🔍 **검토팀** | 각 팀 결과물 검수: 스크린샷 눈검수 → 전체 e2e 회귀 → 커밋 | 커밋·PR |
| ✅ **최종승인자** (메인 세션) | 아이디어 승인(보통 5개, 학습 영역이 겹치지 않게), 방향 결정, 배포 | PR 머지 → Pages 배포 확인 → 부모에게 스크린샷 보고 |

## 표준 라운드 절차 (매번 동일)

1. **브랜치 리셋**: `git fetch origin main && git checkout -B <작업브랜치> origin/main`
2. **정적 서버**: `python3 -m http.server 8777 &` (모든 팀이 공유 — 에이전트에게 "이미 떠 있음, 죽이지 말 것" 명시)
3. **팀 투입**: 병렬 에이전트로. 프롬프트에 반드시 포함:
   - CLAUDE.md(절대 규칙)·DESIGN.md(장난감 방 + **가로모드 우선**) 정독 지시
   - **담당 폴더 밖 수정 금지** (병렬 충돌 방지 — 루트 index.html·README·CLAUDE.md는 검토팀 몫)
   - **git 커밋 금지** (검토팀이 일괄)
   - 자기 검증: 해당 앱 validate + e2e 통과, 콘솔 오류 0, 폰·패드 스크린샷 눈검수
   - 스크린샷은 세션 스크래치패드에 저장하고 경로를 보고
4. **검토**: 보고가 오면 스크린샷을 직접 보고, validate 재실행 후 **팀별로 즉시 커밋·푸시** (한국어 `요약 — 부연` 형식)
5. **최종 회귀**: 전 앱 e2e 일괄 실행 + 루트 홈 스모크(카드 수·깨진 이미지·콘솔 0)
6. **배포**: PR 생성 → 머지 → `pages.yml` 워크플로 success 확인 → 부모에게 대표 스크린샷과 함께 한국어로 보고

## 에이전트 프롬프트에 늘 넣는 기술 정보

- Playwright 모듈: `/opt/node22/lib/node_modules/playwright` (`playwright install` 금지), 크로미움 `/opt/pw-browsers/chromium`
- e2e 명령: 루트에서 `node <앱>/tools/e2e.mjs` (8777 서버), practika만 `BASE_URL=http://127.0.0.1:8777/practika/`, pixel은 `PW_MODULE=... node pixel/tools/e2e.js`
- 기준 화면 3종: 패드 가로 1180×820(1차), 폰 가로 844×390, 폰 세로 390×844
- **잘림 자동 검사** 패턴: `getBoundingClientRect()`가 뷰포트 안 + `scrollHeight <= clientHeight` (선례: 펫 방 검증 스크립트)
- 합성 포인터로 드래그/펜 검증 (`pointerType: 'pen'`은 필기 검사용)

## 자주 쓰는 팀별 지시 요점

- **새 앱 구축**: math/의 골격(헤더 🏠·pet-slot·🗣️·⭐, 스크립트 순서 profile→앱→parent-settings→voice-settings→pet-avatar→pet→time-limit→sw-register)을 복제. manifest+아이콘(SVG→Playwright PNG), `Profile.key('<앱>-playground-v1')`, validate-data.js와 e2e.mjs(새로고침 진행도 유지 검사 필수)를 직접 작성. 루트 홈 카드는 검토팀이 등록
- **콘텐츠 확장**: 기존 id·순서·저장 형식 절대 불변(추가만), 검증기의 최소 개수 계약을 새 기준으로 강화, 시각물은 전수 렌더 눈검수
- **씬(찾기 놀이터) 리디자인**: `data-find`/`data-diff`의 id·label·level·좌표 계약 유지, defs/그라디언트 금지, `validate-scene.js` + `render-scene.js` 렌더 검수
- **펫 관련**: `enjoy-pet-v1` 필드 추가만+마이그레이션, `collection`은 `[{sp,name,at}]` 객체 배열, 모든 학습 활동은 `Pet.awardSnack(1)`(작은 완료)/`Pet.awardMeal(1)`(큰 완료)을 준다는 원칙
- **가로모드**: DESIGN.md "가로모드 우선" 절 — `@media (orientation: landscape)`에서 grid-template-areas 2단 재배치, 세로모드 무변경
