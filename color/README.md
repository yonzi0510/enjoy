# 색칠 놀이터 — Happy Color 스타일 색칠 앱 (로컬 PWA)

## 1. Happy Color 조사 요약

Happy Color®는 "컬러 바이 넘버(color by number)" 앱이다. 핵심 작동 방식:

1. **그림 라이브러리** — 카테고리별 도안 갤러리. 각 도안은 진행률/완성 여부를 썸네일에 표시.
2. **번호 영역 채색** — 그림이 수십 개의 영역(면)으로 나뉘고 각 영역에 팔레트 번호가 표시됨.
   팔레트에서 번호 색을 고르면 **그 번호의 미채색 영역만 진한 회색으로 강조**되어 찾기 쉬움.
3. **탭 채색** — 강조된 영역을 탭하면 그 색으로 채워짐. 다른 번호 영역을 탭하면 채워지지 않음(부드러운 피드백만).
4. **색상별 진행률** — 팔레트 스와치에 진행 링이 차오르고, 그 색을 다 칠하면 ✓ 표시 후
   다음 미완성 색이 자동 선택됨.
5. **확대/이동** — 핀치 줌·드래그 팬(데스크톱은 휠 줌)으로 작은 영역도 정확히 탭.
6. **힌트** — 남은 영역 하나를 찾아 화면을 이동/강조해 줌.
7. **자동 저장** — 도안별 진행 상황이 자동 저장되어 언제든 이어서 칠할 수 있음.
8. **완성 연출** — 마지막 영역을 칠하면 윤곽선·번호가 사라지며 완성 그림 공개 + 축하 연출,
   "내 작품" 갤러리에 보관.

## 2. 로컬 앱 재구축 계획

이 저장소의 기존 서브앱(`play/` 찾기 놀이터, `english/`)과 동일한 방식:
**서버·로그인·외부 라이브러리 없는 정적 PWA** (바닐라 JS + SVG + localStorage).
홈 화면에 추가하면 전체화면 단독 앱처럼 동작하고, 모든 데이터는 기기에만 저장된다.

### 구조

```
color/
  index.html            # 홈(갤러리) · 색칠 화면 · 완성 오버레이 · 내 작품 화면
  manifest.webmanifest  # PWA 설치 (standalone)
  icon-192.png / icon-512.png
  css/style.css
  js/
    progress.js         # localStorage 저장/복원 (도안별 채색 상태·완성 기록)
    audio.js            # WebAudio 효과음 + 한국어 음성 안내 (파일 없음, play/와 동일 방식)
    engine.js           # 갤러리·채색 캔버스·줌/팬·힌트·완성 연출
    pictures/*.js       # 도안 데이터 (그림 1개 = 파일 1개)
  tools/
    validate-picture.js # 도안 계약 정적 검증기 (node)
    e2e.js              # Playwright 처음~끝 자동 플레이 검증
```

### 도안 계약 (pictures/*.js)

```js
PICTURES.push({
  id: 'butterfly', name: '나비', emoji: '🦋', category: 'animal',
  vb: [0, 0, 800, 800],                  // viewBox
  palette: ['#FFD93D', '#FF8FB2', ...],  // 번호 = 배열 순서 + 1
  regions: [                             // 뒤에 오는 영역이 위에 그려짐(레이어)
    { t: 'rect', x, y, w, h, c: 0 },     // c = palette 인덱스
    { t: 'circle', cx, cy, r, c: 1 },
    { t: 'ellipse', cx, cy, rx, ry, c: 2 },
    { t: 'poly', pts: [[x,y],...], c: 3 },
    { t: 'path', d: 'M...Z', c: 4, lx?, ly? },  // lx/ly = 번호 위치 수동 지정(선택)
  ],
});
```

- 번호 위치는 엔진이 **가려지지 않은 내부 점**을 자동 탐색(격자 샘플링 + `isPointInFill`
  + 상위 레이어 가림 검사)해서 배치한다. 완전히 가려진 영역은 검증 실패로 잡아낸다.
- 도안은 JS 파일이므로 반복 요소(꽃잎·비늘·창문 등)는 코드로 생성해도 된다.

### 핵심 기능 구현

| Happy Color 기능 | 구현 |
|---|---|
| 갤러리 + 진행률 | 홈 그리드에 미니 SVG 썸네일(채색된 부분은 색, 나머지는 흰색) + % 뱃지 |
| 번호 영역 강조 | 선택 색 미채색 영역 `#cfcfcf`, 나머지 미채색 `#f4f4f4`, 번호 텍스트 |
| 탭 채색 | pointerup(이동량 작을 때만) → 좌표 → 최상위 포함 영역 판정 → 색 일치 시 채움 |
| 오답 피드백 | 영역이 잠깐 흔들리는 플래시 + 부드러운 "통" 소리 (감점 없음) |
| 팔레트 진행 링 | conic-gradient 진행 링, 완료 시 ✓ + 다음 미완성 색 자동 선택 |
| 줌/팬 | viewBox 조작: 1포인터 드래그=팬, 2포인터=핀치, 휠=커서 기준 줌, 더블탭=확대 |
| 힌트 💡 | 선택 색의 남은 영역으로 뷰 이동 + 펄스 링 + 반짝 효과음 |
| 자동 저장 | 채색할 때마다 localStorage(`color-playground-v1`)에 영역 비트 저장 |
| 완성 연출 | 윤곽·번호 페이드아웃 → 전체 보기 → 폭죽 + 팡파레 + 음성 축하 |
| 내 작품 | 완성 도안 갤러리(완성일 표시), 완성작은 "다시 색칠"로 초기화 가능 |

### 도안 라인업 (v1 — 8개, 4개 카테고리)

동물(나비·물고기·고래), 자연(꽃 화분·버섯집), 풍경(무지개 집), 탈것(로켓), 음식(아이스크림).
도안당 영역 20~60개 · 색 6~10개, 쉬운 것부터 어려운 것까지.

### 실행 방법

- **앱**: 정적 파일이므로 아무 정적 서버로 `color/`를 열면 된다.
  예) `python3 -m http.server -d color 8000` → `http://localhost:8000/`
  GitHub Pages 배포 시 `https://<도메인>/color/` 로 접속, 홈 화면에 추가하면 앱처럼 사용.
- **정적 검증**: `node color/tools/validate-picture.js [도안id]`
- **E2E**: `PW_MODULE=<playwright-core 경로> node color/tools/e2e.js`
  (Chromium 실행 파일은 기본 `/opt/pw-browsers/chromium`, `CHROMIUM` 환경변수로 변경.
   `SHOT_DIR=<폴더>` 지정 시 주요 화면 스크린샷 저장)

### 테스트 계획 (기본 기능이 처음부터 끝까지)

1. **정적 계약 검증** — `node color/tools/validate-picture.js` : 모든 도안의 필드·팔레트
   hex·영역 타입·색 인덱스 범위·모든 색 사용 여부·id 중복을 검사.
2. **E2E 자동 플레이** — `node color/tools/e2e.js` (Playwright + 내장 Chromium):
   - 홈 갤러리 렌더 → 도안 열기 → 모든 영역이 탭 가능한 번호 위치를 갖는지 확인
   - 오답 탭(다른 번호 영역)이 채색되지 않는지 확인
   - 팔레트를 순회하며 **실제 클릭으로 전 영역 채색** → 색상별 ✓/자동 선택 확인
   - 완성 오버레이 등장 확인 → 새로고침 → 홈 100% 뱃지·내 작품 등록 확인
   - 두 번째 도안 일부 채색 → 새로고침 → **이어하기 복원** 확인
   - 휠 줌·드래그 팬 후에도 탭 채색이 정확한지 확인
