# 프랙티카 놀이터 — 구축 계획

AI 튜터와 **말하기로 대화하며 영어를 배우는** 앱 [Praktika(praktika.ai)]의 핵심 기능을
서버·로그인·외부 API 없이 로컬(브라우저)에서 다시 구축한다.

## 1. 원본 앱 조사: Praktika (프랙티카)

Praktika는 AI 아바타 튜터와 **음성 대화**로 언어(주로 영어)를 익히는 앱이다.
조사(praktika.ai, OpenAI 사례연구, 리뷰·헬프센터)로 파악한 핵심 구성:

| 원본 기능 | 설명 |
|---|---|
| 학습 경로 설정 | 목표(여행·일상·커리어)·CEFR 레벨(A1~C1)·관심사를 골라 맞춤 코스 생성 |
| AI 아바타 튜터 | 사람 같은 아바타가 말을 걸고, 사용자의 말투·속도에 맞춰 대화를 이어감 |
| 시나리오 롤플레이 | "카페 주문", "공항 체크인" 같은 상황을 튜터와 역할극으로 연습 |
| 실시간 음성 피드백 | 말하기를 듣고 문법·발음·유창성을 평가해 즉시 교정·칭찬 |
| 어휘 복습 | 배운 핵심 표현을 카드로 모아 다시 말하며 복습 |
| 게임화 | 스트릭(연속일)·XP·젬(보석)·레슨 별점으로 동기 부여 |

핵심 루프: **튜터가 말한다 → 내가 소리내어 답한다 → 평가·피드백 → 대화가 이어진다 → 레슨 완료 보상.**

## 2. 로컬 재구축 방침

기존 `english/`·`hangul/`과 동일: **의존성 없는 바닐라 JS PWA.**
서버·로그인·외부 리소스 없음. 정적 사이트(GitHub Pages)로 배포, 진행도는 localStorage.

- **AI 튜터의 "지능"**은 로컬에선 실제 LLM 대신 **대본(dialogue script) 기반 대화 엔진**으로 재현한다.
  각 레슨은 튜터 대사 + 기대 응답(정답 표현 후보)들로 이뤄진 분기/선형 대본이다.
- **말하기 인식**은 Web Speech API `SpeechRecognition`(en-US), **튜터 목소리**는 `speechSynthesis`
  (튜터=en-US, 힌트=ko-KR), **효과음**은 Web Audio.
- **피드백/채점**은 사용자 음성 텍스트를 기대 표현과 **퍼지 매칭**(정규화 + 토큰/편집거리)해
  0~100점 정확도로 환산하고, 맞은 단어·놓친 단어를 색으로 표시한다.
- 마이크가 없는 환경(헤드리스 테스트 등)에서도 되도록 `window.__simulateSpeech(text)` 훅으로
  인식 결과를 주입할 수 있게 한다(`english/`와 동일 패턴).

## 3. 기능 매핑 (원본 → 로컬 구현)

1. **학습 경로(홈/코스맵)** — 3개 트랙(🧳 여행 · ☕ 일상 · 💼 커리어) × 각 4개 레슨.
   레슨은 순서대로 잠금 해제, 별점(정확도 기반)으로 완료 표시.
2. **대화 세션(핵심)** — 아바타 튜터가 대사를 말하고(TTS+자막+한국어 힌트),
   내가 말할 목표 표현을 보여준다 → 마이크로 말하면 → 퍼지 매칭 채점 →
   맞으면 칭찬하며 다음 턴, 아쉬우면 교정("이렇게 말해봐요")·다시듣기·다시말하기 →
   여러 턴을 거쳐 시나리오를 완주.
3. **실시간 피드백** — 턴마다 정확도 점수 + 단어별 하이라이트(맞음/놓침),
   모범 답안 듣기와 따라 말하기 재시도.
4. **레슨 결과** — 완주 시 평균 정확도로 별 1~3개, XP·젬 지급, 스트릭 갱신, 배운 표현 저장.
5. **어휘 복습** — 레슨에서 배운 핵심 표현을 카드로 모아, 튜터가 뜻/한국어를 주면
   영어로 말해보는 복습 모드(정확도 채점).
6. **게임화 헤더** — 스트릭 🔥 · XP ⭐ · 젬 💎 를 홈 상단에 표시, localStorage 유지.

## 4. 파일 구조

```
practika/
  index.html            화면 골격 (홈/코스맵/세션/결과/복습)
  manifest.webmanifest  PWA 설치용
  icon-192.png icon-512.png  (Playwright로 아바타 SVG 래스터화 생성)
  css/style.css
  js/data/lessons.js    트랙·레슨·대본·어휘 데이터
  js/speech.js          STT(en-US) + TTS(en/ko) + Web Audio 효과음
  js/match.js           퍼지 매칭·정확도 채점(정규화·토큰·편집거리)
  js/progress.js        localStorage: 스트릭·XP·젬·레슨 별점·배운 표현
  js/tutor.js           대화 엔진(대본 진행·응답 평가)
  js/review.js          어휘 복습 모드
  js/app.js             화면 전환·홈·코스맵·세션 UI·결과
  tools/validate-data.js  데이터 검증(Node)
  tools/make-icons.mjs    아이콘 생성(Playwright)
  tools/e2e.mjs           Playwright 종단 테스트
```

## 5. 테스트 계획

- `node tools/validate-data.js` — 트랙·레슨·대본 구조, 각 튜터 턴의 기대 응답 존재,
  어휘 표현 정합성, 힌트/번역 존재를 검증.
- `node tools/e2e.mjs` — 실제 Chromium으로 구동:
  홈 → 코스맵 → 레슨 시작 → 대화 전 턴을 `__simulateSpeech`로 완주(정답/오답 경로 모두 확인)
  → 결과(별·XP·젬) → 스트릭 갱신 → 어휘 복습 1회 → 새로고침 후 진행도 유지 검증.
- 테스트 훅: `window.__practikaTest` 로 현재 화면·현재 기대 표현을 노출해 결정적 테스트.
- `__simulateSpeech(text)` 로 마이크 없이 인식 결과 주입.
